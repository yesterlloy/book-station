# 📖 BookStation 快速启动指南

## 🚀 方式一：一键启动（推荐，Mac/Linux）

```bash
# 赋予执行权限
chmod +x start.sh

# 一键启动所有服务
./start.sh
```

## 🚀 方式二：手动分步启动

### 第一步：启动数据库

```bash
# 在项目根目录执行
docker compose up -d mongodb redis
```

### 第二步：插入测试数据

```bash
cd backend

# 首次运行需要安装依赖
npm install

# 执行数据脚本
node seed-data.js
```

✅ 成功后会显示：
```
✅ 数据库连接成功
🗑️ 清空旧数据完成
✅ 插入 8 本小说
✅ 插入 160 个章节
✅ 更新小说章节数完成

🎉 测试数据插入成功！
```

### 第三步：启动后端服务

```bash
# 在 backend 目录下
npm run dev
```

服务运行在 **http://localhost:6001**

### 第四步：启动前端

使用任意静态文件服务器：

```bash
# 方式 1：Python
cd frontend
python3 -m http.server 6002

# 方式 2：Node.js
npm install -g serve
serve -s frontend -p 6002

# 方式 3：VS Code Live Server
右键 frontend/index.html - Open with Live Server
```

前端运行在 **http://localhost:6002**

## 🐳 Docker 常用命令

```bash
# 查看服务状态
docker compose ps

# 查看日志
docker compose logs -f

# 停止服务
docker compose stop

# 停止并删除容器（保留数据）
docker compose down

# 完全重置（删除数据）
docker compose down -v
```

## 🔧 常见问题

### Q: 提示 MongoDB 连接失败？

A: 确保 Docker 正在运行，并且 MongoDB 容器已启动：
```bash
docker compose ps
```

### Q: 前端显示"加载失败"？

A: 检查后端服务是否正常运行：
```bash
# 测试 API
curl http://localhost:6001/api/novels
```

### Q: 如何重新插入测试数据？

A: 再次执行脚本即可，会自动清空旧数据：
```bash
node backend/seed-data.js
```

### Q: 如何修改数据库配置？

A: 编辑 `.env` 文件：
```env
MONGO_HOST=localhost
MONGO_PORT=27017
REDIS_PORT=6380
```
