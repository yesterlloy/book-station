# 用户角色权限说明

## 角色分类

| 角色 | 说明 | 权限层级 |
|------|------|---------|
| 👑 **admin (管理员)** | 系统最高权限，可管理所有用户和内容 | 3 级（最高） |
| ✍️ **author (作者)** | 可以创作、管理自己的小说和章节 | 2 级 |
| 📖 **reader (读者)** | 普通用户，可阅读、收藏、评论 | 1 级（默认） |

---

## 权限继承规则

权限层级高的角色自动拥有层级低的所有权限：

```
admin (3级)
    └── author (2级)
            └── reader (1级)
```

即：
- **管理员** 拥有所有作者和读者的权限
- **作者** 拥有所有读者的权限

## 身份验证流程

### 1. 注册

**接口**：`POST /api/auth/register`

```javascript
// 请求参数
{
  username: "reader1",        // 用户名
  email: "reader1@xxx.com",   // 邮箱
  password: "password123",     // 密码
  role: "reader"              // 可选，默认 reader，可注册为 author
}
```

**注意**：
- 不允许直接注册 `admin` 角色
- `admin` 只能通过数据库直接创建或由其他管理员授予

### 2. 登录

**接口**：`POST /api/auth/login`

```javascript
// 请求参数
{
  username: "reader1",        // 用户名或邮箱
  password: "password123"      // 密码
}

// 返回
{
  code: 0,
  data: {
    token: "jwt_token_here",
    user: {
      id: "user_id",
      username: "reader1",
      role: "reader",           // 角色信息
      ...
    }
  }
}
```

### 3. 携带 Token 访问

在请求头中添加：
```
Authorization: Bearer <token>
```

## 中间件说明

### 权限中间件

```javascript
const { requireAdmin, requireAuthor, requireLogin, allowRoles } = require('./middleware/permission');

// 需要管理员权限
router.get('/admin/users', requireAdmin, handler);

// 需要作者权限（管理员也可以）
router.post('/author/novels', requireAuthor, handler);

// 需要登录
router.get('/user/profile', requireLogin, handler);

// 自定义角色列表
router.get('/some/path', allowRoles(['admin', 'author']), handler);
```

### 状态检查中间件

```javascript
// 检查用户账号状态（在 auth 中间件之后使用）
router.use(checkUserStatus);
```

用户状态说明：
- `active` - 正常状态
- `banned` - 已封禁，无法登录
- `pending` - 待审核，登录受限

## 安全最佳实践

1. **永远不要**在前端信任 `role` 字段，服务端必须再次验证
2. 管理员账号使用强密码，定期更换
3. 重要操作（如修改角色、删除用户）需要记录日志
4. 敏感操作建议增加二次验证
5. 定期检查作者认证状态，及时处理违规账号
